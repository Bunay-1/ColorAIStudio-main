# Указания за допринасяне (Contributing Guidelines)

Благодарим ви, че се интересувате от подобряването на ICAP Platform!

## Процес на разработка

1.  **Fork** на репозиторието.
2.  Създайте нов **branch** за вашата функция или фикс (`feature/amazing-feature` или `fix/critical-bug`).
3.  Направете вашите промени.
4.  Уверете се, че всички тестове минават (`pytest`).
5.  Обновете документацията и `CHANGELOG.md`, ако е необходимо.
6.  Изпратете **Pull Request**.

## Стандарти за код

- Използвайте **PEP 8** за Python кода.
- Пишете смислени commit съобщения.
- Добавяйте тестове за всяка нова функционалност.
- Всички лог съобщения и низове в интерфейса трябва да са на български език.

# Contributing Guidelines

Thank you for your interest in improving ICAP Platform!

## Development Process

1.  **Fork** the repository.
2.  Create a new **branch** for your feature or fix (`feature/amazing-feature` or `fix/critical-bug`).
3.  Make your changes.
4.  Ensure all tests pass (`pytest`).
5.  Update documentation and `CHANGELOG.md` if necessary.
6.  Submit a **Pull Request**.

## Code Standards

- Use **PEP 8** for Python code.
- Write meaningful commit messages.
- Add tests for every new functionality.
- All system logs and interface strings should be in Bulgarian.
